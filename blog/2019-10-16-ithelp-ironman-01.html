# iT 邦幫忙鐵人賽 DAY 1 前言

Visual Studio Code (opens new window) (簡稱 VS Code) 是由微軟 (opens new window)開發，同時支援 Windows、Linux、macOS 且開放原始碼的程式碼編輯器。時至今日，已經很明顯的打敗了另外兩個競爭對手，Sublime Text (opens new window) 和 Atom (opens new window)，這應該也是目前微軟 (opens new window)數一數二知名的開放原始碼專案。

# 講古時間

想當初大約在 2010 年時，我一開始在剛接觸後端程式語言 PHP 的時候，為了要把程式碼儲存成 UTF-8 without BOM 編碼，所以是使用 Notepad++ (opens new window) 這個編輯器。到了大約 2013 年的時候，偶然之下得知了 Sublime Text (opens new window) 這款編輯器，才知道原來 Notepad++ (opens new window) 也只是比微軟 (opens new window)內建的 Notepad (opens new window) 高一個等級，但是跟 Sublime Text (opens new window) 的距離還是非常遙遠，於是我就決定跳槽了。

但是後來，由於 Sublime Text (opens new window) 版本更新速度過於緩慢，而且在 macOS 中對於中文注音輸入法支援度奇差無比，所以大概在 2015 年的時候跳槽到 Atom (opens new window) 用了一陣子，但是 Atom 在安裝了一堆外掛套件以後，每次重新啟動的速度就會超級慢，於是在 2017 年左右，終於下定決心跳槽到 VS Code (opens new window)，然後就一直用到了現在。

# 為什麼會想寫這個主題

以前被 Sublime Text (opens new window) 多行編輯功能吸引而換用 Sublime Text (opens new window)，但是其實一開始也沒到很熟練，是因為後來偶然之下發現由 Wes Bos (opens new window) 所寫的 Sublime Text Power User (opens new window) 這本書，下定決心把書裡面的內容看完，這才覺得真的有熟練 Sublime Text (opens new window) 這個編輯器，並且在其中學到了不少加速開發的小技巧。也是因為熟悉了這款編輯器，並且為了不要讓它一直跳出免費試用的警告，所以才決定買了 Sublime Text (opens new window) 的授權。

後來，VS Code (opens new window) 出現以後，包含了很多致敬 Sublime Text (opens new window) 的功能，並且還特地支援 Sublime Text (opens new window) 功能的快捷鍵，所以我幾乎也是無痛轉移到 VS Code (opens new window)，2017 年換了人生第二份工作，公司規模比第一份工作大很多，這才發現其實也不是很多人都很精通 VS Code (opens new window)，於是我就興起了閱讀官方文件 (opens new window)，然後稍微把文件整理成中文，然後就可以分享給同事，一起增加開發效率。
